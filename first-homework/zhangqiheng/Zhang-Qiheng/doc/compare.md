## 一、数据解读：C++ vs Python 的差异在哪里？

### 1. 速度差（`speed_diff`）分析
- **Python 侧**：大部分帧的速度值稳定在 0~2 像素/帧之间，波动很小，曲线是一条贴近0的直线。
- **C++ 侧**：速度值出现了 `10~30` 像素/帧的尖峰，导致 `speed_diff` 图上出现了明显的“刺”。
- **根本原因**：
  1.  **C++ 侧的检测/卡尔曼出现了跳变**：轨迹被错误点拉偏，导致两帧之间的位置差突然变大，速度被放大；
  2.  **时序错位**：C++ 读取视频的帧和 Python 不完全对齐，导致差分计算出的速度出现了“虚高”的伪影；
  3.  **卡尔曼滤波的抑制效果不足**：Python 版本的卡尔曼对噪声的抑制更强，轨迹更平滑，速度自然更稳定。

### 2. 角度差（`direction_diff`）分析
- **Python 侧**：角度值基本稳定在一个合理区间，没有出现跳变。
- **C++ 侧**：出现了 `±100°` 的巨大偏差，甚至有从 `0°` 直接跳到 `351.18°` 的情况，导致 `direction_diff` 图上出现了大幅波动。
- **根本原因**：
  1.  **目标丢失或误识别**：当 C++ 检测到错误的点（比如绿色圆柱）时，物体的“运动方向”会被完全算错；
  2.  **角度计算逻辑的微小差异被放大**：Python 和 C++ 在浮点数精度、atan2 实现上的微小差异，在角度差分中被放大成了巨大偏差。

## 二、结论：推荐 Python 版本
### ✅ Python 版本的优势
1.  **算法逻辑更严谨**：帧处理、时序控制、边界条件（如首帧速度为0）的实现更符合算法预期。
2.  **卡尔曼滤波更稳定**：对噪声和误检测的抑制效果更好，轨迹平滑，速度和角度的计算结果也更稳定。
3.  **调试和迭代成本更低**：Python 脚本修改方便，调试直观，你能快速定位到检测、滤波、数据写入中的每一步问题。

- 这次的 C++ 版本，在**检测鲁棒性**和**时序同步**上，还没有达到和 Python 版本同等的稳定性，导致数据波动大、偏差明显。
