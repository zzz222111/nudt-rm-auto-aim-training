# homework_final.cpp 技术文档

## 一、概述

`homework_final.cpp` 是一个基于 OpenCV 的**绿色球体实时检测与运动追踪**程序。该程序从视频文件中逐帧读取图像，利用 HSV 颜色空间分割提取绿色区域，通过几何特征（圆形度、亮度）筛选出目标球体，并使用指数移动平均（EMA）对位置、速度、方向进行平滑滤波，最终将追踪结果可视化并输出为视频文件。

## 四、核心算法详解

### 4.1 绿色区域分割

```cpp
cv::cvtColor(frame, hsv, cv::COLOR_BGR2HSV);
cv::Scalar lowerGreen(35, 60, 50);
cv::Scalar upperGreen(90, 255, 255);
cv::inRange(hsv, lowerGreen, upperGreen, mask);
```

**原理：** 将 BGR 图像转换到 HSV 色彩空间，利用 Hue（色调）通道的绿色区间 `[35°, 90°]` 进行阈值分割。Saturation（饱和度）和 Value（明度）分别设置下限 `60` 和 `50`，过滤掉过暗或褪色的像素。

**HSV 绿色范围选择依据：**
| 参数 | 下界 | 上界 | 说明 |
|------|------|------|------|
| H (色调) | 35 | 90 | OpenCV 中 H ∈ [0,180]，绿色对应约 35°–90° |
| S (饱和度) | 60 | 255 | 排除低饱和度的灰白区域 |
| V (明度) | 50 | 255 | 排除过暗的背景噪声 |

### 4.2 形态学处理

```cpp
cv::Mat kernel = cv::getStructuringElement(cv::MORPH_ELLIPSE, {5, 5});
cv::morphologyEx(mask, maskClean, cv::MORPH_OPEN, kernel);
cv::morphologyEx(maskClean, maskClean, cv::MORPH_CLOSE, kernel);
```

- **开运算（MORPH_OPEN）：** 先腐蚀后膨胀，去除小面积噪点。
- **闭运算（MORPH_CLOSE）：** 先膨胀后腐蚀，填充目标内部空洞。
- **结构元素：** 5×5 椭圆形核，适合圆形目标的边界平滑。

### 4.3 轮廓筛选策略（三级过滤）

#### 第一级：面积过滤
```cpp
if (area < 500.0) continue;
```
排除像素面积小于 500 的噪声区域。

#### 第二级：圆形度过滤
```cpp
double circleArea = CV_PI * radius * radius;
double circularity = area / circleArea;
if (circularity < 0.70) continue;
```

**圆形度（Circularity）** 定义为轮廓面积与其最小外接圆面积之比：
$$\text{circularity} = \frac{A_{\text{contour}}}{A_{\text{enclosing circle}}} = \frac{A_{\text{contour}}}{\pi r^2}$$

| 物体 | 典型圆形度 | 结果 |
|------|-----------|------|
| 绿球 | ≈ 0.73–0.95 | ✅ 保留 |
| 绿色圆柱体 | ≈ 0.45–0.60 | ❌ 排除 |
| 阈值 | 0.70 | — |

#### 第三级：亮度过滤
```cpp
int v = static_cast<int>(hsv.at<cv::Vec3b>(center.y, center.x)[2]);
if (v < 120) continue;
```

提取候选区域中心点的 V 通道值，排除较暗的绿色物体（如绿色圆柱体 V ≈ 90–110，绿球 V ≥ 136）。

> **设计意图：** 在绿球与绿色圆柱体共存的场景中，仅靠颜色阈值无法区分。绿球具有两个可区分的物理特征——**更高的圆形度**（球体投影为圆，圆柱体投影为矩形/椭圆）和**更高的表面亮度**（光照反射特性），因此采用圆形度 + 亮度双重筛选。

#### 最优选择
```cpp
if (circularity > bestCirc) {
    bestCirc = circularity;
    bestCenter = center;
    bestRadius = radius;
}
```
在通过所有过滤条件的候选区域中，选择圆形度最高的作为最终目标，确保绿球始终优先于其他绿色干扰物。

### 4.4 指数移动平均（EMA）平滑

系统对三个状态量分别进行 EMA 平滑，消除帧间抖动：

| 状态量 | 平滑系数 | 公式 |
|--------|----------|------|
| 球心位置 | α_pos = 0.25 | $P_t = 0.25 \cdot p_t + 0.75 \cdot P_{t-1}$ |
| 速度大小 | α_spd = 0.30 | $S_t = 0.30 \cdot s_t + 0.70 \cdot S_{t-1}$ |
| 运动方向 | α_dir = 0.20 | $D_t = 0.20 \cdot d_t + 0.80 \cdot D_{t-1}$（归一化后） |

**平滑系数选择原则：**
- **位置 (0.25)：** 适中的平滑度，在追踪响应速度与稳定性之间取得平衡。
- **速度 (0.30)：** 稍快的响应，速度变化需及时反映（对自瞄系统尤为关键）。
- **方向 (0.20)：** 最平滑，方向变化应缓慢连续，避免箭头剧烈摆动。

> 方向向量平滑后需重新归一化：$D_t \leftarrow \frac{D_t}{\|D_t\|}$，确保始终为单位向量。

### 4.5 速度计算与可视化

```cpp
cv::Point2f delta = smoothedCenter - prevCenter;
float rawSpeed = std::hypot(delta.x, delta.y);
```

基于连续两帧平滑位置的欧氏距离计算速度（单位：像素/帧）。

**速度箭头颜色映射（按速度分段线性插值）：**

| 速度范围 | 颜色 | RGB 值 |
|----------|------|--------|
| < 10 px/f | 绿色（慢速） | (0, 255, 0) |
| 10–30 px/f | 绿色 → 红色 渐变 | (0, G→0, R→255) |
| > 30 px/f | 红色（快速） | (0, 0, 255) |

**箭头长度控制：**
```cpp
float arrowLen = std::min(smoothedSpeed * arrowScale, maxArrowLength);
```
箭头长度 = 速度 × 3.0，上限 150 像素，防止高速时箭头超出画面。

### 4.6 轨迹记录

```cpp
std::deque<cv::Point2f> trail;
const int maxTrailLength = 30;
```

使用双端队列保存最近 30 帧的平滑位置。绘制时采用**渐变颜色映射**——越旧的点越偏绿（起点色），越新的点越偏红（末端色），实现类似"彗尾"的视觉效果：

$$\text{color}(i) = \left(0,\ 255 \cdot \left(1 - \frac{i}{n}\right),\ 255 \cdot \frac{i}{n}\right)$$

## 五、状态机设计

```
                        检测到目标
    ┌──────────┐     ───────────────▶    ┌──────────┐
    │  丢失状态  │                        │  追踪状态  │
    │ (LOST)    │ ◀──────────────────── │ (TRACKING) │
    └──────────┘     连续丢失目标         └──────────┘
```

**丢失目标时的重置逻辑（第 202–208 行）：**
```cpp
prevCenter = cv::Point2f(-1, -1);
smoothedCenter = cv::Point2f(-1, -1);
smoothedSpeed = 0.0f;
smoothedDir = cv::Point2f(0.0f, 0.0f);
trail.clear();
```

重置所有状态变量，确保目标重新出现时从零开始平滑，避免前一追踪周期的残留状态污染新追踪。

## 九、适用场景与局限

### 适用场景
- 单一绿色球体在简单背景下的运动追踪
- 需要区分球体与其他同色非圆形物体（如圆柱体）的场景
- 要求输出平滑运动参数（位置、速度、方向）的视觉系统

### 局限性
1. **单目标假设：** 只追踪一个绿色物体，无法处理多球场景。
2. **颜色硬编码：** 绿色 HSV 范围固定，光照剧烈变化时可能失效。
3. **无遮挡处理：** 目标短暂遮挡后状态完全重置，无预测/重捕获机制。
4. **无尺度不变性：** 球体大小变化（远近移动）时追踪正常，但无尺寸自适应平滑。
5. **PC 平台绑定：** 依赖 OpenCV VideoIO 的 MP4 编解码器，嵌入式平台需修改。

## 十、扩展方向

1. **卡尔曼滤波替代 EMA：** 引入运动模型，提高遮挡场景下的预测能力。
2. **自适应颜色阈值：** 基于目标区域直方图动态调整 HSV 范围。
3. **多目标追踪：** 采用匈牙利算法 + 卡尔曼滤波实现多球同时追踪。
4. **深度学习替代方案：** 使用 YOLO/SSD 检测器替代手工特征，提升复杂背景下的鲁棒性。
5. **实时性能优化：** 引入 ROI 预测缩小搜索区域，降低每帧计算量。
