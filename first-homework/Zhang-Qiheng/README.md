# Week1作业说明文档
**张起恒**

## 1. 概述

程序核心功能清单：
1. 区分绿色小球与绿色圆柱干扰；
2. 目标包围识别框、椭圆轮廓、中心点、自适应长度运动箭头、实时速度/运动状态文字；
3. 逐帧导出 CSV 数值文件，记录帧索引、时间、瞬时速度、运动角度，用于算法误差定量对比；
4. 输出完整标注视频，支持本地任意目录迁移运行，JSON配置统一管理全部可调参数。
5. 使用python、cpp两种实现方法，并用diff.py比对实现结果。

## 6. 核心实现方案
### 6.1 绿色前景提取
1. 将原图 BGR 色彩空间转换为 HSV，通过配置文件内置上下限阈值分离绿色区域；
2. 采用开运算+闭运算形态学滤波：先腐蚀消除细小噪点，再膨胀还原目标完整轮廓，去除画面零散绿色像素干扰。

### 6.2 多维度几何特征筛选（区分小球/圆柱）
对所有绿色轮廓分层过滤：
1. 面积过滤：剔除过小噪点、超大色块；
2. 椭圆长宽比约束：标准球形长宽比接近1，圆柱长条轮廓直接过滤；
3. 轮廓径向标准差约束：完美圆形各点到中心距离均匀，圆柱轮廓离散度更高；
4. 边缘自适应阈值：小球贴画面边界、轮廓被裁切时，自动放宽长宽比、离散度筛选标准，避免贴边直接丢失目标。

### 6.3 时空联合择优匹配（抑制目标跳变）
传统仅靠形状打分容易在小球、圆柱间切换，本程序新增时序约束：
1. 使用前一帧跟踪中心与运动速度，预测当前帧目标合理位置；
2. 候选轮廓与预测位置距离过远直接舍弃，拒绝远距离干扰物体；
3. 轮廓半径突变过大直接过滤，抑制尺寸跳变带来的速度异常峰值；
4. 综合形状分数、运动连续性双重指标选出最优跟踪目标。

### 6.4 卡尔曼滤波 + 短时遮挡预测
1. 正常检测到小球：卡尔曼滤波更新观测值，平滑坐标，消除抖动；
2. 短时遮挡（未超过配置最大丢失帧数）：启用匀速外推预测，持续输出合理目标坐标，CSV持续输出有效速度，不会直接归零；
3. 长时间完全丢失：重置卡尔曼状态，清空轨迹缓存，等待重新识别小球。

### 6.5 运动可视化与量化数据输出
1. 目标绘制：绿色包围识别框、椭圆轮廓、红色中心点；遮挡预测帧额外标注 `green ball predicted` 文字区分；
2. 运动矢量：根据相邻帧坐标差计算速度矢量，箭头长度随瞬时速度自适应缩放；同时打印实时像素速度；
3. 轨迹缓存：缓存历史N帧坐标，绘制粉色连续运动轨迹线；
4. 量化输出：逐帧写入CSV，存储帧索引、绝对时间、瞬时速度、0~360°运动角度，可用于和C++版本做误差对比绘图。

## 7. 关键问题与解决方案
### 7.1 绿色圆柱干扰，跟踪来回跳变
**问题现象**：画面内同时存在绿色小球与绿色圆柱，仅靠颜色分割会随机切换跟踪对象。
**处理方案**：
1. 引入椭圆长宽比、轮廓径向标准差区分球形与柱形；
2. 择优阶段增加时序距离惩罚，远离上一帧运动轨迹的轮廓直接淘汰；
3. 双重打分机制，优先保证跟踪连续性，而非仅看轮廓形状完美度。

### 7.2 小球贴近画面边缘时跟踪丢失
**问题现象**：小球移动到画面左右、上下边界，一半轮廓被画面裁切，几何特征变形，严格筛选规则直接丢弃目标，跟踪中断。
**处理方案**：
1. 增加边界判定函数，识别贴边轮廓；
2. 贴边目标自动放宽长宽比、轮廓离散度筛选阈值；
3. 不直接丢弃边缘轮廓，允许参与择优匹配，实现“出框半遮挡仍可识别跟踪”。

### 7.3 小球被物体短时遮挡，速度直接归零失真
**问题现象**：小球被手、障碍物短暂挡住，检测不到轮廓，瞬时速度置0，数据断层，运动轨迹断开。
**处理方案**：
1. 丢失帧计数，短时遮挡区间启用匀速外推预测坐标；
2. 预测坐标作为有效跟踪点参与速度计算与可视化，遮挡阶段持续输出符合运动规律的数值；
3. 画面标注预测状态，区分真实检测/预测推算两种跟踪结果。

### 7.4 速度曲线存在大量异常跳变尖峰
**问题现象**：目标匹配错误、轮廓尺寸突变时，相邻帧坐标剧烈偏移，速度数值突变，箭头长度、方向剧烈抖动。
**处理方案**：
1. 时序约束过滤远距离干扰候选，杜绝跨物体跳变；
2. 轮廓半径突变过大直接舍弃，过滤尺寸异常轮廓；
3. 卡尔曼滤波平滑坐标，抑制单帧噪声带来的速度抖动。

### 7.5 项目更换电脑/文件夹路径后程序找不到配置、视频
**问题现象**：剪切项目文件夹、更换磁盘目录运行，相对路径失效，报配置文件、视频不存在。
**处理方案**：
1. 全部路径以 `main.py` 脚本自身绝对路径为基准自动换算；
2. JSON内仅保留相对路径，代码运行时自动拼接项目根目录，全平台兼容正反斜杠；
3. 输出目录自动递归创建，无需手动新建文件夹，任意目录迁移均可直接运行。
