# Python+OpenCV 实现绿球视觉识别与跟踪系统

## 项目简介

本代码基于 Python 与 OpenCV 计算机视觉库，完成视频流中绿色球体的较精准检测、实时跟踪及运动状态可视化。核心采用 HSV 颜色空间分割技术提取绿色目标区域，结合轮廓几何特征过滤非球体干扰物；通过计算相邻帧目标位移获取运动速度，辅以短时位置预测机制，保障目标短暂丢失时跟踪的连续性；同时将运动方向、速度大小、帧序号、运动状态等信息直观标注在视频中。

## 主要功能

-   **目标检测**：通过设定 HSV 绿色阈值范围分割出绿色像素区域，再依据轮廓的面积、圆度、长宽比等几何特征，过滤掉背景干扰与非球体目标，实现绿球的精准识别。
-   **目标跟踪**：计算绿球在相邻视频帧中的像素位移量，换算为运动速度；当目标短时遮挡时，基于历史运动速度预测目标位置并完成帧间匹配，大幅提升跟踪的稳定性。
-   **可视化标注**：
    
    -   用圆形轮廓标记绿球的实际位置，蓝点标注球心坐标；
    -   箭头标注运动方向，箭头长度与运动速度正相关（速度越快，箭头越长）；
    -   实时叠加文字信息：当前帧序号、绿球实时运动速度、运动状态（加速 / 减速 / 匀速）；
    -   目标状态标注：目标丢失时在视频左上角标注 “Target Lost”。

## 可调参数说明

-   `HSV 绿色阈值`：`LOWER_GREEN` 为颜色下限，`UPPER_GREEN` 为颜色上限；数组三个数值依次对应 H（色相）、S（饱和度）、V（明度），由 cv2.inRange 生成颜色掩码。调试原则：识别不到绿球 → 适当放宽范围；误判其他绿色物体 → 收窄范围。
-   `轮廓形状筛选参数（排除圆柱、方块干扰）`：

    - `MIN_AREA`：有效轮廓的最小面积，单位为像素；作用：过滤面积过小的噪点、碎块，只保留尺寸足够的目标。远处小球被漏检 → 减小数值；细碎噪点被误识别 → 增大数值

    - `ASPECT_RATIO_RANGE`：轮廓外接矩形的宽高比（宽度 / 高度）允许范围；作用：正圆形宽高比接近 1，竖长圆柱、长条物体的宽高比会明显偏离 1，以此过滤非球形目标。透视变形、倾斜视角下的球被过滤 → 放宽范围（如 0.6~1.4）；圆柱、方块仍被误判 → 收窄范围（如 0.8~1.2）

    - `MIN_CIRCULARITY`：最小圆形度阈值，计算公式为 圆形度 = 4 × π × 面积 / 周长²，完美正圆的圆形度为 1，形状越不规则数值越低；作用：精准筛选圆形目标，排除方形、多边形、不规则色块（正方形圆形度约为 0.785）。边缘不光滑的球被过滤 → 降低阈值（如 0.6）；非圆形物体仍被误判 → 提高阈值（如 0.8）
    
-   `可视化参数`：

    -`ARROW_SCALE`：运动箭头的长度缩放系数；计算逻辑：箭头终点偏移量 = 帧间位移像素数 × 缩放系数。        箭头太短不直观 → 增大系数；箭头太长超出画面 → 减小系数
