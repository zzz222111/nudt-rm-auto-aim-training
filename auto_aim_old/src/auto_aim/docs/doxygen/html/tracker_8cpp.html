Detailed Description

Concrete tracker logic: selection, EKF initialization and update, and state-machine for lost/detecting/tracking states.

The tracker keeps a target identity (string id) and applies the EKF for smoothing pose/velocity/yaw/radius estimates. It attempts to match incoming detections to the predicted position, applies safeguards for sudden jumps, and enforces simple limits for radius/velocity values depending on the detected armor class.
